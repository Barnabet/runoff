# Runoff v1.1 — Run-over-Run Awareness + Trust-Layer Hardening

**Approved:** 2026-07-18 (brainstormed with Louis; diff style = figure deltas; continuity = stability contract).
**Scope:** two tracks. Track 2: the Reader diffs a run against its predecessor and the model receives the prior document as continuity context. Track 3: audit strictness, answer timing, and failed-section surfacing. No schema migration — the previous document is read from the predecessor's `run_completed` event.

Out of scope (v1.2, separately brainstormed): projects, project/blueprint memory system, builder copilot agent.

---

## 1. Previous-run resolution (`@runoff/core`)

New file `packages/core/src/db/previousRun.ts`, exported from the core index:

```ts
import type BetterSqlite3 from "better-sqlite3";
import type { RunDocument } from "../types/document.js";

export interface PreviousRun {
  runId: string;
  completedAt: string; // the predecessor run's created_at (ISO from SQLite)
  document: RunDocument;
}

export function previousCompletedDocument(
  sqlite: BetterSqlite3.Database,
  blueprintId: string,
  current: { runId: string; createdAt: string },
): PreviousRun | null;
```

Behavior:
- Predecessor = `SELECT id, created_at FROM runs WHERE blueprint_id = ? AND status = 'complete' AND id != ? AND created_at <= ? ORDER BY created_at DESC, id DESC LIMIT 1` (ties on `created_at` broken by `id DESC`; `<=` plus `id != current` handles same-second runs).
- Its document comes from `SELECT payload FROM run_events WHERE run_id = ? ORDER BY seq` — find the `run_completed` event, return its `document`. If no such event exists (corrupt row), return `null`.
- No predecessor → `null`. Callers treat `null` as "first run": no continuity block, no diff UI.

Used by the worker (feeds the engine) and the web queries (feeds the Reader) so the lookup cannot drift.

## 2. Continuity in the engine (stability contract)

- `executeRun` opts gain `previousDocument?: RunDocument`.
- For each **auto/review** section, the orchestrator resolves `previousDocument?.sections.find((s) => s.key === section.key)` and passes its `blocksToPlainText` as `previousSectionText` to `draftSection`. Fixed sections skip the model and are unaffected. A section with no same-key predecessor gets no block.
- `draftSection` args and `sectionUserPrompt` args gain `previousSectionText?: string`. The prompt block renders between the completed-sections block and the steers block, with this exact wording:

```
Last run's version of this section (keep its structure and wording where the
underlying data is unchanged; update figures and note material changes):
<text>
```

- Worker `processOne` calls `previousCompletedDocument` with the claimed run's blueprint id / run id / created_at and passes the document into `executeRun`.
- `scripts/eval.ts` passes no `previousDocument` (unchanged behavior).

## 3. Figure-delta diff (`@runoff/core` + Reader)

### 3a. Shared numeric parsing

`stripNumeric` moves from `packages/engine/src/checks.ts` to core as `parseFigure(text: string): number` (strip `$`, `,`, `%`, then `parseFloat`). `checks.ts` imports it; behavior unchanged.

### 3b. Diff function

New file `packages/core/src/diff.ts`, exported from the core index:

```ts
export interface FigureDelta {
  sectionKey: string;
  sourceId: string;
  locator: string;
  before: number;
  after: number;
}
export type SectionDiffStatus = "new" | "removed" | "changed" | "unchanged";
export interface RunDiff {
  deltas: FigureDelta[];
  sections: Record<string, SectionDiffStatus>;
}
export function diffRuns(current: RunDocument, previous: RunDocument): RunDiff;
```

Rules:
- Sections match by `key`. `sections` covers the union: current-only → `new`; previous-only → `removed`; both → `blocksToPlainText` equality decides `unchanged` vs `changed`.
- Within a matched section, cited spans (paragraphs and table cells) key by `${sourceId}|${locator.trim()}`; the **first** occurrence per key per side wins.
- A delta is emitted when a key exists on both sides, both spans parse via `parseFigure` to finite numbers, and the values differ. Zero-difference and non-numeric pairs are dropped.

### 3c. Reader UI

- `getRunPayload` gains `previous: PreviousRun | null` via `previousCompletedDocument` (only meaningful for terminal runs; live runs may return whatever the query finds — the Live page ignores it).
- `ReaderView` computes `diffRuns(document, previous.document)` client-side when `previous` is non-null and the run has a document.
- **Inline badges:** `SectionBlocks` gains optional `figureDeltas?: Record<string, { before: number; after: number }>` keyed `${sourceId}|${locator.trim()}` — the same normalization `diffRuns` uses, and `SectionBlocks` trims when looking up. When a cited span's key matches, a small badge renders after the `CitationChip`: mono, 9.5px, `no-print`; `▲ <abs diff>` in ink/60 when after > before, `▼ <abs diff>` in pencil when after < before (values via `toLocaleString("en-US")`).
- **Rail card:** new `DeltaCard` component in the Reader rail, above the flags: header `SINCE LAST RUN — <previous completedAt as short date>`, one row per delta (`<source label> · <locator>` left, `<before> → <after>` right, mono), and a summary line `N changed · M unchanged[ · K new][ · J removed]` (zero-count segments omitted). Entire card `no-print`. No previous run, or zero deltas AND all sections unchanged → card not rendered.
- Print output is unchanged (badges and card carry `no-print`).
- Live Run page is untouched.

## 4. Trust-layer hardening (engine)

### 4a. Audit strictness (`checks.ts`, inside `auditCitations`)

Evaluation order per cited figure-bearing span, each failure with a pinned prefix:

1. Unbound source (existing): `figure cites unbound source <id>: <fig>`.
2. **New — locator/citation source mismatch:** if the locator matches `AGG_REF` and its embedded sourceId ≠ `span.citation.sourceId` → `locator source mismatch: cites <citation sourceId> but locator references <locator sourceId>`; skip recompute for this span.
3. **New — unverifiable aggregate locator:** if the locator matches `AGG_REF` but `computeAgg` throws (unknown source, column, or filter column) → `unverifiable locator: <locator>` (replaces today's silent `continue`).
4. Recompute mismatch (existing): `citation mismatch: <fig> vs computed <n>`.

Non-`AGG_REF` locators (quote references) remain exempt from 2–4.

### 4b. Answers reach the retry (`run.ts`)

`await drainInputs()` immediately before the retry `draftSection` call, so an answer (or steer) posted while the first draft was being written/checked influences the redraft. Existing section-boundary drain unchanged.

### 4c. Failed-section surfacing

- Reducer: the per-section record gains `error?: string`, set by `section_failed` (state already becomes `"failed"`; log line unchanged).
- `LiveSection`: new `failed` branch (and new `error?: string` prop, passed from the projection by `LiveRunView`): instead of the typing caret, render an italic serif note in pencil red — `Drafting failed — <error>. The run continued without this section.`
- `scripts/eval.ts` console IO: print `[section_failed] <sectionKey> — <error>`.

## 5. Testing

- **core** `previousRun.test.ts`: returns latest complete predecessor (skips queued/failed/newer runs, excludes self, same-second tie via id); null when none; null when `run_completed` event missing.
- **core** `diff.test.ts`: changed figure → delta with before/after; equal values dropped; non-numeric cited text dropped; first-occurrence-wins; section statuses new/removed/changed/unchanged; table-cell citations participate.
- **engine** `prompts.test.ts`: pinned continuity wording present when `previousSectionText` given, absent otherwise.
- **engine** `run.test.ts`: with `previousDocument`, the fake client's received prompt contains the prior section text for the matching key and none for a new key; drain-before-retry — an answer queued in `pollInputs` after the first draft appears in the retry prompt's answered-questions block.
- **engine** `checks.test.ts`: pinned prefixes for `locator source mismatch: ` and `unverifiable locator: ` (unknown column, unknown source, unknown filter column); quote locators still pass silently.
- **core** `reducer.test.ts`: `section_failed` sets `error` on the section record.
- **web** `runView.test.tsx`: failed section renders the pencil note, not a caret.
- **web** `reader.test.tsx`: delta badge renders next to a changed cited figure and not next to unchanged ones; DeltaCard rows and summary; card absent when `previous` is null; badge/card elements carry `no-print`.
- **web** `runsApi.test.ts` or `api.test.ts`: `getRunPayload.previous` populated for a second completed run, null for the first.

## Global constraints

- No DB schema changes; no migration.
- New audit failure prefixes are load-bearing (they flow into retry feedback and tests pin them): `locator source mismatch: `, `unverifiable locator: `.
- Continuity prompt wording is pinned exactly as written in §2.
- Printed documents must not change: all new Reader UI is `no-print`.
- All existing tests keep passing; the eval script must still complete a live run.
